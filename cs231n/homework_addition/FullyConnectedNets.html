Fully-Connected Neural Nets¶

In the previous homework you implemented a fully-connected two-layer neural network on CIFAR-10. The implementation was simple but not very modular since the loss and gradient were computed in a single monolithic function. This is manageable for a simple two-layer network, but would become impractical as we move to bigger models. Ideally we want to build networks using a more modular design so that we can implement different layer types in isolation and then snap them together into models with different architectures.

In this exercise we will implement fully-connected networks using a more modular approach. For each layer we will implement a forward and a backward function. The forward function will receive inputs, weights, and other parameters and will return both an output and a cache object storing data needed for the backward pass, like this:

上述代码对x的形状重新组织，变成一个batch_size行的矩阵。-1可以自动转变为列数，就是不管下面维度有多少都变成一行数据。
接下来需要对输入数据加权变成输出：

out = np.dot(x.reshape(len(x), -1), w)

`np.dot`的操作对两个矩阵进行内积操作。
假设展开后x的列数为c，两个矩阵形状为(batch_size, c)和(c, output_dim)。
相乘后得到形状为(batch_size, output_dim)的矩阵。

SVM Loss: $$ \begin{align} L_i &= \sum_{j\neq y_i}max(0, s_j-s_{y_i}+1)\\ L &= \frac{1}{N}\sum_{i=0}^{N} L_i \end{align} $$

Softmax Loss: $$ \begin{align} L_i &= -log(\frac{e^{s_{y_i}}}{\sum_j e^{s_j}})\\ &=-(log(e^{s_{y_i}})-log(\sum_j e^{s_j}))\\ &=-(s_{y_i}-log(\sum_j e^{s_j}))\\ L &= \frac{1}{N}\sum_{i=0}^{N} L_i \end{align} $$

SVM损失函数的实现

根据前面的公式计算SVM损失值。 

correct_class_scores = x[np.arange(N), y]

取出每个样本全连接层输出中对应正确标签位置的值。 

correct_class_scores[:, None]

把`correct_class_scores`扩展为2维的矩阵，使一个值和x矩阵的一行数据对应 

margins = np.maximum(0, x - correct_class_scores[:, None] + 1.0)

每个score值减去正确位置的score值后加1，比0小的都变成0。之后求和再求均值，得到总体的损失值。
函数还要返回对输入x（score值）的求导。实际上只有比正确score小的那些score指对导数有贡献，其导数为1。
而正确标签对应的score则由于出现错误score时被减，因此其导数为那一行数据中错误的score的个数。

Softmax损失函数的实现

根据上面公式，可以计算出输入的score每个对应的softmax值。之后再取正确标签对应的那个softmax值，然后对所有数据点做一次均值。 反向求导过程，仍然区分非正确标签的score的求导和正确标签score的求导。 对于非正确标签的score $s_j$ $$ds_j = \frac{e^{s_j}}{\sum_{k=0}^{classes\_num}e^{s_k}}$$ 对于正确标签的score $s_y$ $$ds_y = -1 + \frac{e^{s_y}}{\sum_{k=0}^{classes\_num}e^{s_k}}$$

[FILL THIS IN] The five layer net seenmed more sensitive to the initialization scale. To explain that, I assume all the weights are initalized with 0.
五层的网络对初始化权重大小比较敏感。
For the three layer net,
对于如下图所示三层的网络：
![](http://oxsdrqmze.bkt.clouddn.com/18-6-20/28453934.jpg) the gradient is like:
梯度的计算如下所示：
$$ dW_3 = \frac{\partial Loss}{\partial W_3} = h_2 \\ dW_2 = \frac{\partial Loss}{\partial h_2}\frac{\partial h_2}{\partial W_2} = W_3 · h_1 \\ dW_1 = \frac{\partial Loss}{\partial h_2}\frac{\partial h_2}{\partial h_1}\frac{\partial h_1}{\partial W_1}= W_3 · W_2 · x $$ Clearly, the gradient will stay 0 forever if the weights are 0.
But if the weights have some little scale, then the value will accumulate along the iterations, and the weights can get chance to become greater.
The only hope to trigger the accumulation is on the third equation, for the others are related with hidden layers, which should be calculated through more layers of weights than the input layer x.
The scale of weight just have to great enough to let $dW_1$ become non-zero, which make the $W_1$ in next iteration non-zero, causing a series of accumulation between forward and backward, then all the weights gain non-zero values.
For the five layer net, the gradient is like:
如果全部的权重初始化为0，那么网络不会迭代，所有层的数据都会是0，反向求导得到的梯度也会一直是0。但是如果权重能稍微大一点，也就是给权重设置一个大于非0的方差，就有可能使网络中有流通的值。准确来说，只要权重的初始大小能够达到使`dW_1`不是0，那么权重就会开启更新，逐渐累加。 $$ dW_5 = \frac{\partial Loss}{\partial W_5} = h_4 \\ dW_4 = \frac{\partial Loss}{\partial h_4}\frac{\partial h_4}{\partial W_4} = W_5 · h_3 \\ dW_3 = \frac{\partial Loss}{\partial h_4}\frac{\partial h_4}{\partial h_3}\frac{\partial h_3}{\partial W_3} = W_5 · W_4 · h_2 \\ dW_2 = \frac{\partial Loss}{\partial h_4}\frac{\partial h_4}{\partial h_3}\frac{\partial h_3}{\partial h_2}\frac{\partial h_2}{\partial W_2} = W_5 · W_4 · W_3 · h_1 \\ dW_1 = \frac{\partial Loss}{\partial h_4}\frac{\partial h_4}{\partial h_3}\frac{\partial h_3}{\partial h_2}\frac{\partial h_2}{\partial h_1}\frac{\partial h_1}{\partial w_1} = W_5 · W_4 · W_3 · W_2 · x \\ $$ To cause the accuulation as the three layers net does, the initialization of weights scale should great enough to make $dW_1$ non-zero. So it is hard for the five layers net to train, and it is more sensitive to the initialization weight scales.

动量法是为了解决局部最优和鞍点(某些方向梯度向上，某些方向梯度向下)问题而出现的。大概做法是在随着梯度调整权重的基础上，权重再加上之前梯度的滑动平均。形象地看，把梯度下降看作小球在凹凸不平的碗中滚动，我们知道小球会朝最底部滚动。原来的梯度下降方法，相当于认为小球没有惯性，滚动到凹槽或平地就停止了。而事实上小球具有惯性，到平地会继续往前滚动，这个惯性是之前加速度积累起来的。 $$ v_{t+1} = \rho v_{t} + \alpha \nabla f(x_t)\\ w_{t+1} = w_{t} + v_{t} $$ 其中$\rho$是一个较大的比例，如0.9，$\alpha$则为学习率。

RMSprop是基于动态调整学习率的动机提出来的。一般认为训练到梯度比较小的时候，也就是比较平缓的区域时，需要较大的学习率，同时迭代久了，接近最优解的时候需要降低学习率。于是有人提出了AdaGrad方法，学习率除以之前每次迭代的梯度的平方和，但是这会导致这个除数越来越大，学习率越来越小。RMSprop通过将历史梯度平方和进行滑动平均的方法，给了学习率重新变大的机会。 $$ W^2_{t+1} = \rho W^2_{t} + (1-\rho) \nabla f(w_{t+1}) * \nabla f(w_{t+1})\\ w_{t+1} = w_{t} - \frac{\alpha}{W^2_{t+1}} * \nabla f(w_{t+1}) $$

Inline Question 3:¶

AdaGrad, like Adam, is a per-parameter optimization method that uses the following update rule:

cache += dw**2
w += - learning_rate * dw / (np.sqrt(cache) + eps)

John notices that when he was training a network with AdaGrad that the updates became very small, and that his network was learning slowly. Using your knowledge of the AdaGrad update rule, why do you think the updates would become very small? Would Adam have the same issue?

Answer:¶

Because the 'cache' is monotone increasing through iteration. In the update equation, the denominator will become larger and larger, w will change more and more slowly. There is no chance that the step can become larger again. As for Adam, thanks to the 'beta2', cache become the moving average of dw\*dw, which leave a chance for taking large step after many iteration. AdaGrad的方法每次都是除以历史梯度的平方和，这个值随着迭代次数是单调递增的，因此导致学习率会一路下降，最后网络不再迭代学习。Adam方法里融合了RMSprop方法，学习率除以的是历史梯度的平方和的滑动平均，这个值不会单调递增，学习率也有机会重新变大。
